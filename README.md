# 跳舞吧

**跳舞吧** 是一个面向中老年用户的 iOS 风格舞团工具型应用 MVP。它的目标不是提供复杂的社区运营功能，而是用尽可能少的步骤帮助用户完成三件最核心的事情：**开始跳舞、找到附近舞团、通过语音快速找地点**。

当前版本已经按照老年友好方向进行了多轮收敛。首页采用“大主按钮 + 两个辅助入口 + 大标签底部导航”的结构，尽量减少信息密度，降低理解成本，并保留真实数据交互所需的基础状态管理与后端接口层。

| 项目项 | 说明 |
|---|---|
| 项目名称 | 跳舞吧 |
| 产品定位 | 面向中老年用户的广场舞快速组织与找队工具 |
| 当前阶段 | iOS App MVP |
| 核心原则 | 字大、按钮大、操作短路径、低学习成本 |
| 技术栈 | Expo Router、React Native、TypeScript、TanStack Query、tRPC |
| 数据策略 | 本地持久化 + MVP 后端路由，便于后续切换正式数据库 |

## 当前已实现的核心能力

当前版本重点围绕老人用户最容易理解的主路径进行构建。用户可以直接在首页点击“开始跳舞”，系统将按流程执行定位、拍照、附近舞团判断，并完成加入或创建舞团的 MVP 交互。若用户不想直接开始，也可以先进入“附近舞团”查看周边队伍，或通过“说话找地”走语音找场地流程。

| 功能模块 | 当前状态 | 说明 |
|---|---|---|
| 首页极简入口 | 已完成 | 超大“开始跳舞”按钮，辅以“附近舞团”“说话找地” |
| 附近舞团列表 | 已完成 | 按距离展示舞团，支持进入详情 |
| 开始跳舞流程 | 已完成 | 支持开始、强制新建、查看结果与继续跳转 |
| 语音找地入口 | 已完成 | 已保留入口与承接流程 |
| 我的页面 | 已完成 | 支持头像与昵称维护 |
| 分享卡片 | 已完成 | 支持基础分享动作 |
| 本地数据持久化 | 已完成 | 用于保存个人状态与舞团交互结果 |
| MVP 后端路由 | 已完成 | 用于承载真实交互接口的后续扩展 |

## 适老化设计方向

这个项目的设计重点不是信息展示，而是**一眼会用**。因此界面上尽量避免复杂卡片嵌套、难懂文案和小点击区域。所有关键页面都优先采用单列布局、大字号、短语义标签和高对比按钮，确保用户不需要阅读很久就能理解下一步该做什么。

| 设计要求 | 当前策略 |
|---|---|
| 大按钮 | 首页主按钮与关键操作按钮均扩大热区 |
| 大字号 | 标题、按钮、状态信息采用更大的字号层级 |
| 低信息密度 | 首页只保留三类核心动作 |
| 明确层级 | 红色主操作、浅底信息区、橙色辅助操作 |
| 可返回 | 关键流程页补充返回路径，避免用户迷路 |

## 主要页面结构

项目当前围绕几个关键页面组织。首页负责快速进入主路径；附近舞团页负责查看和选择附近队伍；开始跳舞页负责执行主流程并展示结果；我的页面负责保存最基本的个人资料。这样的结构适合 MVP 阶段快速验证真实需求，同时也便于后续向正式产品演进。

| 页面 | 作用 |
|---|---|
| 首页 | 进入开始跳舞、附近舞团、说话找地 |
| 附近舞团 | 查看附近舞团并进入详情 |
| 开始跳舞 | 定位、拍照、判断并创建/加入舞团 |
| 消息 | 预留提醒与消息中心 |
| 我的 | 编辑头像、昵称与基础资料 |
| 舞团详情 | 查看单个舞团的当前信息 |
| 分享卡片 | 生成并分享结果卡片 |

## 本地运行方式

如果你要在本地继续开发，可以按下面的方式启动项目。默认开发体验由 Expo 与本地 API 服务共同组成。

```bash
pnpm install
pnpm dev
```

开发过程中也可以单独执行检查与测试：

```bash
pnpm test
pnpm check
```

## 项目目录说明

为了让后续功能继续扩展，项目已经按前端页面、共享逻辑、服务端路由和测试分层组织。

| 目录 | 说明 |
|---|---|
| `app/` | Expo Router 页面与路由 |
| `components/` | 复用 UI 组件 |
| `lib/` | 前端状态、工具函数、业务逻辑 |
| `server/` | tRPC 路由、服务端能力与数据层 |
| `tests/` | Vitest 自动化测试 |
| `assets/` | 图标、启动图与静态资源 |
| `scripts/` | 项目初始化与批量改写脚本 |

## 下一步建议

当前版本已经适合继续做第二轮产品打磨。下一步最值得优先推进的方向，是把“语音找地”和“开始跳舞”进一步连成一条无障碍主路径，再补充真正的登录体系与正式数据库持久化，让用户状态、舞团状态和分享行为都能跨设备保存。
